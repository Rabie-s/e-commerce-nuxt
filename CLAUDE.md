## Project Overview

This is a Nuxt 4 e-commerce application in early development stages. The project uses Tailwind CSS v4 for styling.

## Development Commands

```bash
# Install dependencies
npm install

# Start development server (runs on http://localhost:3000)
npm run dev

# Build for production
npm run build

# Preview production build locally
npm run preview

# Generate static site
npm run generate
```

## Architecture

### Tech Stack
- **Framework**: Nuxt 4.4.2 with Vue 3.5.30
- **Styling**: Tailwind CSS 4.2.2 via Vite plugin
- **Routing**: File-based routing via Vue Router 5.0.4
- **TypeScript**: Full TypeScript support configured

### Project Structure
```
app/
  ├── app.vue          # Root application component
  └── pages/           # File-based routing
      └── index.vue    # Home page
```

### Key Patterns
- **File-based routing**: Pages in `app/pages/` automatically become routes
- **Auto-imports**: Components, composables, and utilities are auto-imported
- **Styling**: Tailwind CSS v4 via `@tailwindcss/vite` plugin (no CSS import needed in config)
- **Nuxt 4 compatibility**: Uses compatibility date `2025-07-15`

### Configuration Notes
- Nuxt config is in `nuxt.config.ts`
- Tailwind CSS v4 configured via Vite plugin - automatically injects styles
- DevTools enabled for development
- TypeScript configuration auto-generated by Nuxt
- Tailwind v4 customization: create CSS files with `@import` directives (e.g., `@import "tailwindcss/theme"`)

## Backend API Integration

This frontend connects to a Laravel e-commerce backend located at `C:\Users\ASUS\Desktop\e-commerce-laravel`.

### API Configuration
- **Base URL**: `http://localhost:8000/api` (default Laravel port)
- **API Version**: `/api/v1/`
- **Authentication**: Laravel Sanctum (Bearer tokens)
- **Auth Header**: `Authorization: Bearer {token}`

### Available Endpoints

#### Products
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/products` | GET | Get paginated products (12 per page) |
| `/api/v1/products/{id}` | GET | Get single product with variants |

**Query Parameters** (for listing):
- `category_id` - Filter by category
- `brand_id` - Filter by brand
- `search` - Search by product name

#### Categories
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/categories` | GET | Get category tree (hierarchical) |
| `/api/v1/categories/{id}` | GET | Get specific category |

#### Brands
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/brands` | GET | Get all brands |
| `/api/v1/brands/{id}` | GET | Get specific brand |

#### Orders
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/orders` | POST | Create new order |
| `/api/v1/orders/{uuid}` | GET | Get order by UUID |

### Data Models

#### Product Structure
```typescript
{
  id: number
  name: string
  description: string
  status: boolean
  main_image: string
  images: Image[]
  category: { id: number, name: string }
  brand: { id: number, name: string }
  default_variant: ProductVariant
  variants: ProductVariant[]
}

type ProductVariant = {
  id: number
  sku: string
  price: number
  is_default: boolean
  stock: number
  main_image: string
  attribute_values: {
    id: number
    value: string
    type: { id: number, name: string }
  }[]
}
```

#### Category Structure
```typescript
{
  id: number
  name: string
  main_image: string
  children: Category[]  // Hierarchical - can be nested
}
```

#### Order Creation Request
```typescript
{
  customer_info: {
    phone_number: string
    first_name: string
    last_name: string
    city: string
    address: string
    nearby_landmark?: string
  }
  items: {
    variant_id: number
    quantity: number
  }[]
}
```

#### Order Response
```typescript
{
  order_uuid: string
  order_id: number
  status: 'Pending' | 'Confirmed' | 'Shipped' | 'Delivered' | 'Cancelled'
  total_price: number
  customer: { /* customer info */ }
  items: { /* order items */ }
  payment: {
    method: 'Cod'
    status: 'Pending' | 'Collected' | 'Failed' | 'Refunded'
    amount: number
  }
  created_at: string
}
```

### Pagination Format
Product listings return paginated responses:
```typescript
{
  data: Product[]
  links: { first: string, last: string, prev: string | null, next: string | null }
  meta: {
    current_page: number
    from: number
    last_page: number
    per_page: number
    to: number
    total: number
  }
}
```

### Validation Rules
- Phone number: max 20 characters
- First/last name: max 255 characters
- City: max 255 characters
- Address: max 500 characters
- Nearby landmark: optional, max 255 characters
- Quantity: min 1, max 999999
- Stock validation: backend checks availability before creating order

### Error Responses
- **422**: Validation error (returns `errors` object with field-specific messages)
- **404**: Resource not found
- **500**: Server error

## Frontend Implementation Requirements

Based on the backend API, the frontend needs to implement:

1. **Product Listing** (`/products`)
   - Display paginated product grid
   - Filter by category and brand
   - Search functionality
   - Show product card with image, name, price

2. **Product Detail** (`/products/[id]`)
   - Show all product information
   - Variant selection (color, size, etc.)
   - Stock display
   - Add to cart functionality

3. **Shopping Cart** (client-side state)
   - Use Pinia for cart state management
   - Store selected variants and quantities
   - Calculate totals
   - Persist to localStorage

4. **Checkout** (`/checkout`)
   - Customer information form
   - Order review
   - Submit order to backend

5. **Order Confirmation** (`/orders/[uuid]`)
   - Display order details
   - Show order status and payment info

6. **Category Navigation**
   - Display hierarchical category tree
   - Filter products by category
